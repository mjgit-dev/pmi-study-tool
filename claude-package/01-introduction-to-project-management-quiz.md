# Introduction to Project Management — Quiz

## Project Life Cycle

## Practice Questions

Q1. A project manager has just confirmed that the organization has a valid business need and that the proposed solution is achievable within available resources. Which phase of the project life cycle has the project manager just completed activities for?

A) Organizing and Preparing
B) Carrying Out the Work
C) Closing the Project
D) Starting the Project

**Answer: D** — Identifying the business need and confirming feasibility are the two primary activities of the **Starting the Project** phase. A) is wrong because organizing and preparing focuses on creating the detailed project management plan, not feasibility assessment. B) is wrong because carrying out the work involves execution and monitoring. C) is wrong because closing involves formalizing acceptance of the product and ending the project.

---

Q2. A project team has a fully approved project management plan and is now producing deliverables. The project manager notices that actual costs are beginning to exceed what was planned. Which best describes what the project manager should be doing during this phase of the life cycle?

A) Formalizing stakeholder acceptance of the product
B) Assessing the feasibility of the project
C) Executing the plan while simultaneously monitoring progress
D) Creating the detailed project management plan

**Answer: C** — The team is in the **Carrying Out the Work** phase, which involves both executing the plan and monitoring progress. The fact that actual costs are diverging from planned costs is exactly the kind of issue monitoring is meant to catch. A) is a closing activity. B) is a starting-phase activity. D) belongs to the organizing and preparing phase, which has already been completed.

---

Q3. A software project has reached the point where the client has reviewed and accepted the final deliverable, all project documentation has been archived, and the project team has been released. Which phase of the project life cycle does this scenario describe, and what is its primary purpose?

A) Carrying Out the Work — to execute the project management plan
B) Closing the Project — to formalize acceptance and bring the project to an orderly end
C) Organizing and Preparing — to ensure all documentation is in place
D) Starting the Project — to confirm business need and feasibility

**Answer: B** — All of the activities described — client acceptance of the deliverable, archiving documentation, and releasing the team — are hallmarks of the **Closing the Project** phase, whose purpose is to formalize product acceptance and end the project in an orderly way. A) is wrong because execution is the work production phase, not the wrap-up. C) is wrong because organizing and preparing is about planning, not finalizing. D) is wrong because starting focuses on need identification and feasibility.

---

## Flashcards

- **Project Life Cycle** — The sequence of phases that a project passes through from start to finish, providing a basic framework for managing the project regardless of the specific work involved.

- **Starting the Project (Phase 1)** — The first phase of the project life cycle, in which the project manager identifies the business need and confirms that the project is feasible.

- **Organizing and Preparing (Phase 2)** — The second phase of the project life cycle, focused on creating a detailed project management plan that guides all subsequent execution.

- **Carrying Out the Work (Phase 3)** — The third phase of the project life cycle, in which the team executes the project management plan while simultaneously monitoring progress against it.

- **Closing the Project (Phase 4)** — The final phase of the project life cycle, in which the product is formally accepted by the relevant stakeholders and the project is brought to an orderly end.

- **Feasibility (in Starting Phase)** — An assessment conducted during the starting phase to determine whether the project is realistic, achievable, and worth pursuing before resources are committed.

- **Formalized Acceptance** — A key activity of the closing phase in which stakeholders officially accept the product, service, or result delivered by the project, signaling the project's completion.

## What is a Project

## Practice Questions

**Q1.** A manufacturing company runs a process where the same consumer product is assembled on a production line every day with no planned end date. A new initiative is launched to redesign the packaging of that product, with a target completion date six months away. Which of the following correctly classifies these two activities according to PMI's framework?

A) Both are projects because they both involve effort and resources.
B) The daily assembly is a project; the packaging redesign is an operation.
C) The daily assembly is an operation; the packaging redesign is a project.
D) Both are operations because the company performs them within the same organization.

**Answer: C** — The daily assembly is repetitive and ongoing with no defined end, which makes it an operation. The packaging redesign is temporary (six-month timeline) and produces a unique result (new packaging design), which makes it a project. Option A is wrong because effort and resource use alone do not define a project. Option B reverses the correct classification. Option D is wrong because organizational setting does not determine whether something is a project or operation.

---

**Q2.** At the beginning of a software development project, the project manager creates a high-level plan with broad milestones because detailed requirements have not yet been gathered. As user interviews are completed over the following weeks, the plan is updated with specific feature lists, timelines, and resource assignments. A stakeholder raises concern that the plan "keeps changing." How should the project manager best explain this situation?

A) The project is experiencing scope creep and the change control process should be initiated.
B) The project is following progressive elaboration, which is an expected characteristic of projects.
C) The project manager failed to plan adequately at the start and is now correcting mistakes.
D) The project should be placed on hold until a complete and final plan can be produced.

**Answer: B** — Progressive elaboration is the process by which the project plan becomes more detailed as more information becomes available, and it is explicitly recognized by PMI as a key characteristic of projects. Option A is wrong because scope creep involves unauthorized changes to scope, not the natural refinement of detail. Option C is wrong because incomplete detail at project initiation is normal and expected, not a planning failure. Option D is wrong because waiting for perfect information before starting would paradict the concept of progressive elaboration entirely.

---

**Q3.** A project manager is assigned to oversee the construction of a pedestrian bridge in a city. The bridge, once built, will serve the public for an estimated 75 years. A junior team member argues that because the bridge will stand for 75 years, the work cannot be considered "temporary" and therefore is not a project. How should the project manager respond?

A) The team member is correct — the long lifespan of the deliverable means this is operations, not a project.
B) The team member is incorrect — "temporary" refers to the project effort, not the lifespan of the deliverable.
C) The team member is partially correct — the project becomes an operation once construction exceeds five years.
D) The team member is correct — PMI classifies any work lasting more than one year as an operation.

**Answer: B** — PMI explicitly clarifies that "temporary" applies to the project itself (the construction effort), which has a defined start and end, not to the product it creates. The bridge lasting 75 years is irrelevant to the classification of the construction work as a project. Option A misapplies the definition of temporary. Option C introduces a fabricated five-year rule that does not exist in PMI's framework. Option D is entirely incorrect — project duration has no bearing on whether work is a project or an operation.

## Flashcards

- **Project** — A temporary endeavor undertaken to create a unique product, service, or result; defined by having a definite beginning and end.

- **Operations** — Ongoing, repetitive work with no defined end date; contrasted with projects, which are temporary and unique.

- **Temporary (in project context)** — Having a defined start and finish; refers to the project effort itself, not the longevity of the deliverable produced.

- **Unique Deliverable** — A product, service, or result that has not been produced in exactly that form before; one of the three key characteristics of a project.

- **Progressive Elaboration** — The process by which a project plan becomes increasingly detailed and refined as more information becomes available throughout the project lifecycle; an expected and normal characteristic of projects.

- **Project Management** — The application of knowledge, skills, tools, and techniques to project activities to meet project requirements (PMI's official definition).

- **Knowledge, Skills, Tools, and Techniques** — The four categories of capability that PMI identifies as what project managers apply to project activities in order to meet project requirements.
