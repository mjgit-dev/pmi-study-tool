# Project Life Cycle

## Overview of the Project Life Cycle

**Core Definition:** "Every project passes through distinct phases called the project life cycle."

The project life cycle is the universal framework that governs how any project is structured and managed, regardless of the industry, size, or nature of the work involved. Because it applies to all projects, it gives project managers a consistent way to think about progress, control, and completion.

**Important Concept:** The life cycle provides a basic framework for managing the project regardless of the specific work involved — meaning these phases apply whether you are building a skyscraper, launching software, or organizing an event.

**Bottom line:** The project life cycle is the foundational structure that every project follows, giving managers a reliable framework independent of what the project actually produces.

---

## The Four Phases of a Typical Project Life Cycle

A typical project life cycle consists of exactly four phases, each with a distinct purpose and set of activities:

1. **Starting the Project** — The initiation phase where the reason for the project is established and its viability is confirmed.
   - Project managers identify the business need driving the project.
   - The team ensures the project is feasible before committing significant resources.

2. **Organizing and Preparing** — The planning phase where detailed groundwork is laid before execution begins.
   - The team creates a detailed project management plan.
   - This phase translates the approved project concept into a concrete, actionable roadmap.

3. **Carrying Out the Work** — The execution and monitoring phase where the actual project deliverables are produced.
   - The team executes the plan developed in the previous phase.
   - Progress is actively monitored to ensure the project stays on track.

4. **Closing the Project** — The formal conclusion of the project.
   - Acceptance of the product or deliverable is formally recorded.
   - The project is brought to an orderly end, ensuring nothing is left unresolved.

**Exam Tip:** Know all four phase names and their defining activities. PMI expects you to recognize what happens in each phase and in what sequence — starting → organizing and preparing → carrying out the work → closing.

**Bottom line:** The four phases of the project life cycle move a project from identifying a need all the way through delivering and formally closing the work in a logical, sequential progression.

---

## Phase 1: Starting the Project

During the starting phase, the project manager's primary responsibilities center on two critical questions: *Why are we doing this?* and *Can we actually do it?*

- **Business Need Identification** — The project manager identifies and documents the specific business need or problem the project is intended to address. Without a clear business need, there is no justification for the project.
- **Feasibility Assessment** — The team evaluates whether the project is achievable given available resources, technology, time, and budget constraints. A project that is not feasible should not proceed to planning.

**Critical Point:** Feasibility must be confirmed before moving forward — committing to planning and execution on an infeasible project wastes organizational resources and sets the team up for failure.

**Bottom line:** The starting phase exists to validate that the project is both necessary and achievable before any significant investment of time or money is made.

---

## Phase 2: Organizing and Preparing

The organizing and preparing phase is where strategic intent is translated into tactical planning. The central output of this phase is the **project management plan** — a detailed, comprehensive document that guides all subsequent work.

- The project management plan is described as *detailed*, signaling that surface-level planning is insufficient; every aspect of execution, monitoring, and control should be anticipated and documented here.
- This phase bridges the gap between the approved project concept (from phase one) and the actual execution work (in phase three).

**Key Distinction:** Organizing and preparing is distinct from carrying out the work — planning must be completed before execution begins, not simultaneously or after the fact.

**Bottom line:** The organizing and preparing phase produces the detailed project management plan that serves as the authoritative guide for all execution activities that follow.

---

## Phase 3: Carrying Out the Work

The carrying out phase is where the project management plan is put into action and the project's deliverables are actually produced. This phase has two simultaneous, inseparable activities:

- **Executing the Plan** — The project team performs the work defined in the project management plan, using the resources, schedules, and methods established during planning.
- **Monitoring Progress** — As execution proceeds, the project manager tracks performance against the plan to detect variances early and take corrective action if necessary.

**Important Concept:** Execution and monitoring happen together during this phase — you cannot simply execute without watching for problems, and monitoring without execution produces nothing.

**Bottom line:** Carrying out the work is the phase where planned activities become real deliverables, with constant monitoring ensuring the project stays aligned with the plan.

---

## Phase 4: Closing the Project

The closing phase brings the project to a formal and orderly conclusion. Two key activities define this phase:

- **Formalizing Acceptance** — The customer, sponsor, or relevant stakeholder officially accepts the product, service, or result the project has produced. This acceptance is documented and signals that the deliverable meets the agreed-upon requirements.
- **Orderly Ending** — The project is wound down in a structured way, ensuring that contracts are closed, resources are released, lessons learned are captured, and no loose ends remain.

**Critical Point:** The word *orderly* is significant — a project should never simply stop or be abandoned. Even if a project is terminated early, closing procedures should still be followed to properly document outcomes and release resources.

**Exam Tip:** PMI emphasizes that closing is a required phase, not an optional one. Formalizing acceptance is a key output of closing and confirms that the project has fulfilled its purpose.

**Bottom line:** The closing phase ensures the project ends with formal stakeholder acceptance and a clean, documented conclusion rather than simply stopping when the work appears done.
