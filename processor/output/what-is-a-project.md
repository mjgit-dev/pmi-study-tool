# What is a Project — PMP Certification Lecture Notes

## Definition of a Project

**Core Definition:** "A project is a temporary endeavor undertaken to create a unique product, service, or result."

This definition comes directly from PMI and is the foundational concept underlying all of project management. Every element of this definition carries weight: "temporary" means it has boundaries in time, "endeavor" implies intentional effort, and "unique product, service, or result" means no two projects are exactly alike.

**Exam Tip:** PMI's exact wording — "temporary endeavor undertaken to create a unique product, service, or result" — should be memorized verbatim. The exam may present variations of this definition and ask you to identify the correct one.

**Bottom line:** A project is defined by its temporariness and the uniqueness of what it produces.

---

## Temporary Nature of Projects

Every project has a **definite beginning and end**. This is non-negotiable in PMI's framework — if something does not have a defined start and finish, it is not a project by definition.

- **Temporary** does not mean short. A project to build a bridge may last a decade, but it still has a start date and a completion date.
- The end of a project is reached when its objectives have been met, when it becomes clear the objectives cannot or will not be met, or when the need for the project no longer exists.

**Key Distinction:** Temporary refers to the project itself, not necessarily the deliverable it produces. A bridge built by a project lasts for generations — but the project that created it had a definite end.

**Bottom line:** All projects, regardless of size or duration, must have a defined beginning and a defined end.

---

## Projects vs. Operations

**Key Distinction:** Projects are **temporary and unique**, while **operations** are **ongoing and repetitive**.

| Projects | Operations |
|----------|------------|
| Temporary — has a defined end | Ongoing — continues indefinitely |
| Produces a unique deliverable | Repetitive, standardized output |
| Example: Developing a software application | Example: Running payroll every two weeks |
| Example: Organizing a conference | Example: Manufacturing the same product daily |
| Example: Building a bridge | Example: Maintaining a bridge after construction |

Understanding this distinction is critical because it determines which management framework applies. Once a project ends and its deliverable becomes routine work, that work transitions into operations.

**Exam Tip:** The exam will test whether you can distinguish between project work and operational work. If an activity is repetitive and ongoing, it is operations — not a project.

**Bottom line:** The line between a project and operations is defined by temporariness and uniqueness — operations are neither.

---

## Key Characteristics of a Project

The instructor identified three core characteristics that define a project:

1. **Temporary** — The project has a finite lifespan with a clear beginning and end. It does not go on indefinitely.
   - Example: Organizing a conference has a start (planning begins) and an end (the conference concludes).

2. **Unique Deliverable** — Every project produces something that has not been produced in exactly that form before.
   - Example: Building a bridge is unique because even if similar bridges exist, this specific bridge — with its location, specifications, and stakeholders — has never been built before.
   - Example: Developing a software application produces a unique system tailored to specific requirements.

3. **Progressive Elaboration** — The project plan becomes more detailed as more information becomes available over time.
   - At the start of a project, information is incomplete. As the team learns more, the plan is refined and filled in with greater precision.
   - Example: Early in a conference-planning project, you may only know the general date and theme. As the project progresses, speaker lists, venue details, and schedules become defined.

**Important Concept:** Progressive elaboration is not a flaw or a sign of poor planning — it is an expected, built-in feature of how projects unfold. PMI recognizes that you cannot know everything at the outset.

**Exam Tip:** If an exam question describes a situation where plans are being updated and refined as new information emerges, that is progressive elaboration — not scope creep or poor control.

**Bottom line:** All three characteristics — temporary, unique deliverable, and progressive elaboration — must be understood together as what makes something a project.

## Examples of Projects

The instructor provided concrete examples to illustrate the range and variety of projects:

1. **Building a bridge** — a physical infrastructure project with clear tangible deliverables and a defined completion point.
2. **Developing a software application** — a technology project producing a unique digital product built to specific requirements.
3. **Organizing a conference** — an event-based project with a definite end (the conference itself) and a unique outcome each time it is held.

These examples demonstrate that projects exist across industries and disciplines. The principles of project management apply equally whether the deliverable is physical, digital, or experiential.

**Bottom line:** Projects span every industry and deliverable type, but they all share the same defining characteristics regardless of domain.

## Definition of Project Management

**Core Definition:** "Project management is the application of knowledge, skills, tools, and techniques to project activities to meet project requirements."

This is PMI's official definition and breaks down into four components:

1. **Knowledge** — Understanding of project management concepts, frameworks, and domain expertise.
2. **Skills** — The interpersonal and technical abilities a project manager applies throughout the project.
3. **Tools** — Instruments and methods used to plan, track, and control project work (e.g., scheduling software, risk registers).
4. **Techniques** — Systematic procedures applied to accomplish activities (e.g., critical path method, earned value analysis).

All four components are applied to **project activities** with one ultimate purpose: to **meet project requirements**. This means project management is goal-directed — it is not process for its own sake, but a means to deliver what was promised.

**Exam Tip:** PMI's definition of project management is frequently tested. Know all four elements — knowledge, skills, tools, and techniques — and that they are applied to meet project requirements.

**Bottom line:** Project management is the disciplined application of four categories of capability — knowledge, skills, tools, and techniques — aimed at satisfying project requirements.

## Practice Questions

**Q1.** A manufacturing company runs a process where the same consumer product is assembled on a production line every day with no planned end date. A new initiative is launched to redesign the packaging of that product, with a target completion date six months away. Which of the following correctly classifies these two activities according to PMI's framework?

A) Both are projects because they both involve effort and resources.
B) The daily assembly is a project; the packaging redesign is an operation.
C) The daily assembly is an operation; the packaging redesign is a project.
D) Both are operations because the company performs them within the same organization.

**Answer: C** — The daily assembly is repetitive and ongoing with no defined end, which makes it an operation. The packaging redesign is temporary (six-month timeline) and produces a unique result (new packaging design), which makes it a project. Option A is wrong because effort and resource use alone do not define a project. Option B reverses the correct classification. Option D is wrong because organizational setting does not determine whether something is a project or operation.

---

**Q2.** At the beginning of a software development project, the project manager creates a high-level plan with broad milestones because detailed requirements have not yet been gathered. As user interviews are completed over the following weeks, the plan is updated with specific feature lists, timelines, and resource assignments. A stakeholder raises concern that the plan "keeps changing." How should the project manager best explain this situation?

A) The project is experiencing scope creep and the change control process should be initiated.
B) The project is following progressive elaboration, which is an expected characteristic of projects.
C) The project manager failed to plan adequately at the start and is now correcting mistakes.
D) The project should be placed on hold until a complete and final plan can be produced.

**Answer: B** — Progressive elaboration is the process by which the project plan becomes more detailed as more information becomes available, and it is explicitly recognized by PMI as a key characteristic of projects. Option A is wrong because scope creep involves unauthorized changes to scope, not the natural refinement of detail. Option C is wrong because incomplete detail at project initiation is normal and expected, not a planning failure. Option D is wrong because waiting for perfect information before starting would paradict the concept of progressive elaboration entirely.

---

**Q3.** A project manager is assigned to oversee the construction of a pedestrian bridge in a city. The bridge, once built, will serve the public for an estimated 75 years. A junior team member argues that because the bridge will stand for 75 years, the work cannot be considered "temporary" and therefore is not a project. How should the project manager respond?

A) The team member is correct — the long lifespan of the deliverable means this is operations, not a project.
B) The team member is incorrect — "temporary" refers to the project effort, not the lifespan of the deliverable.
C) The team member is partially correct — the project becomes an operation once construction exceeds five years.
D) The team member is correct — PMI classifies any work lasting more than one year as an operation.

**Answer: B** — PMI explicitly clarifies that "temporary" applies to the project itself (the construction effort), which has a defined start and end, not to the product it creates. The bridge lasting 75 years is irrelevant to the classification of the construction work as a project. Option A misapplies the definition of temporary. Option C introduces a fabricated five-year rule that does not exist in PMI's framework. Option D is entirely incorrect — project duration has no bearing on whether work is a project or an operation.

## Flashcards

- **Project** — A temporary endeavor undertaken to create a unique product, service, or result; defined by having a definite beginning and end.

- **Operations** — Ongoing, repetitive work with no defined end date; contrasted with projects, which are temporary and unique.

- **Temporary (in project context)** — Having a defined start and finish; refers to the project effort itself, not the longevity of the deliverable produced.

- **Unique Deliverable** — A product, service, or result that has not been produced in exactly that form before; one of the three key characteristics of a project.

- **Progressive Elaboration** — The process by which a project plan becomes increasingly detailed and refined as more information becomes available throughout the project lifecycle; an expected and normal characteristic of projects.

- **Project Management** — The application of knowledge, skills, tools, and techniques to project activities to meet project requirements (PMI's official definition).

- **Knowledge, Skills, Tools, and Techniques** — The four categories of capability that PMI identifies as what project managers apply to project activities in order to meet project requirements.
