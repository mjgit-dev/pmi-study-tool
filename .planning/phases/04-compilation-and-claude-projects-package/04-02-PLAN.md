    Create `compiler.js` at project root. This is a synchronous CLI tool that mirrors processor.js structure.

    **Full implementation spec:**

    ```javascript
    'use strict';

    const fs = require('node:fs');
    const path = require('node:path');
    const { parseFrontmatter } = require('./compiler/frontmatter');
    const { extractBody, splitNotesAndQuiz } = require('./compiler/sections');
    const { groupBySectionName, orderSections, toSlug } = require('./compiler/grouper');
    const { buildSectionFile, sectionFilename, buildHandbook } = require('./compiler/builder');
    const { buildSystemPrompt } = require('./compiler/system-prompt');

    const INPUT_DIR = path.join(__dirname, 'processor', 'output');
    const OUTPUT_DIR = path.join(__dirname, 'claude-package');
    ```

    `compileAll(inputDir, outputDir, flags)` — Synchronous function:
    1. Read all `.md` files from inputDir via `fs.readdirSync().filter(f => f.endsWith('.md')).sort()`
    2. If no .md files found, print `'No .md files found in ' + inputDir` and return `{ sections: 0, lectures: 0, files: [] }`
    3. For each file, read content with `fs.readFileSync`, call `parseFrontmatter(content)`. If null, print warning `'Warning: skipping ' + filename + ' (no frontmatter)'` and skip.
    4. For each parsed file, call `extractBody(content)`, then `splitNotesAndQuiz(body)`. Build lecture object: `{ filename, lectureTitle: fm.lectureTitle, sectionName: fm.sectionName, processedAt: fm.processedAt, notes, quiz }`
    5. Call `groupBySectionName(lectures)` then `orderSections(groups)`
    6. If `flags.dryRun`:
       - Print `'Dry run — would compile:'`
       - For each section in order: `'  Section: {name} ({N} lectures)'`
       - Print `'\nWould write:'`
       - For each section: print notes and quiz filenames
       - Print `'  claude-package/handbook.md'`
       - Print `'  claude-package/CLAUDE_INSTRUCTIONS.md'`
       - Return `{ sections: orderedSections.length, lectures: lectures.length, files: [] }`
    7. If not dry run:
       - If outputDir exists, print `'Overwriting existing claude-package/'` and remove with `fs.rmSync(outputDir, { recursive: true, force: true })`
       - Create outputDir with `fs.mkdirSync(outputDir, { recursive: true })`
       - Track `filesWritten = []`
       - For each section (enumerate with index for numbering, 1-based):
         - Build notes file: `buildSectionFile(sectionName, sectionLectures, 'notes')`
         - Build quiz file: `buildSectionFile(sectionName, sectionLectures, 'quiz')`
         - Get filenames: `sectionFilename(index + 1, sectionName, 'notes')` and `sectionFilename(index + 1, sectionName, 'quiz')`
         - Write both files to outputDir
         - Push filenames to filesWritten
         - Progress: `'[{padded}/{total}] Compiling: {sectionName}... done'` matching processor.js padding style (`String(i + 1).padStart(String(total).length, ' ')`)
       - Build and write handbook: `buildHandbook(orderedSections, groups)` to `handbook.md`
       - Build and write system prompt: `buildSystemPrompt()` to `CLAUDE_INSTRUCTIONS.md`
       - Push `'handbook.md'` and `'CLAUDE_INSTRUCTIONS.md'` to filesWritten
       - Print summary: `'\nCompiled: {sections} sections | {lectures} lectures | {filesWritten.length} files written'`
       - Return `{ sections: orderedSections.length, lectures: lectures.length, files: filesWritten }`

    CLI block (inside `if (require.main === module)`):
    - Parse `--dry-run` from `process.argv.slice(2)` — same pattern as processor.js
    - Check `fs.existsSync(INPUT_DIR)` — if not found, print error and `process.exit(1)`
    - Call `compileAll(INPUT_DIR, OUTPUT_DIR, flags)`
    - Export: `module.exports = { compileAll };`
