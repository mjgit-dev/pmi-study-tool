### Integration Gap: EXTR-03 Processor Guard

The 300-word low-word-count threshold is enforced as a UI warning toast in `extractor.js` at extraction time. The flag (`isLowWordCount`) is not written to the JSON output file. If a low-word-count transcript is manually placed in `transcripts/`, the processor will process it without warning.

**Severity:** Low — the guard exists where it matters most (at extraction time, before the file is saved). The processor gap would only affect manually placed files.
**Affects:** EXTR-03 (integration dimension)

### E2E Flow

**Flow: Extract → Process → Compile → Upload**

```
User clicks bookmarklet on Udemy lecture
  → extractor.js scrapes DOM, cleans transcript, downloads {title}.json
  → User places JSON in transcripts/
  → node processor/processor.js transcripts/
  → manifest.js checks processing-state.json (skip if complete)
  → buildMessages() constructs Anthropic API payload
  → client.messages.create() returns notes + questions + flashcards
  → buildMarkdown() prepends YAML frontmatter → writes processor/output/{title}.md
  → node compiler.js
  → parseFrontmatter() extracts lectureTitle, sectionName, processedAt
  → splitNotesAndQuiz() splits at ## Practice Questions
  → groupBySectionName() + orderSections() groups by section, chronological order
  → buildSectionFile() + sectionFilename() writes NN-{slug}-notes.md, NN-{slug}-quiz.md
  → buildHandbook() writes handbook.md with linked ToC
  → buildSystemPrompt() writes CLAUDE_INSTRUCTIONS.md
  → claude-package/ is upload-ready for Claude Projects
```

**Status: COMPLETE — no breaks**
